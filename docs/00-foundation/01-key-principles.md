# Key principles

1. Utility first: features must solve real player pain points.
2. Low conflict: behave like a safe overlay, especially in large modlists.
3. On-demand by default: no always-on cloud calls.
4. Local-first posture: keys and policy live outside the game process.
5. Short outputs: concise, structured, non-disruptive.
6. Strict grounding when facts matter: help answers must rely on provided docs only.
7. Debuggable by design: traceability, replay bundles, versioning.
8. Cost control: caching, throttling, hard budgets, daily spend caps.
9. Graceful degradation: clear user messaging and safe fallbacks.

(These principles are elaborated in the original concept document in this folder.)
